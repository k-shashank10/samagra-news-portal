Table schema:

create table public."NewsData" (
  id bigint generated by default as identity not null,
  created_at timestamp with time zone not null default now(),
  category text null,
  title text null,
  description text null,
  source text null,
  content text null,
  ai_summary text null,
  ai_title text null,
  url text null,
  constraint News Data_pkey primary key (id),
  constraint unique_news unique (title, source)
) TABLESPACE pg_default;

Display rule: always show ai_title if it is not null/empty; if ai_title is
null or empty, fall back to title. Apply the same fallback for
content: prefer ai_content, fall back to content. Never show both.

=====================================================
NAVIGATION
=====================================================

Bottom navigation bar (persistent, 4 tabs): Feed, Explore, Saved, Account.
  - Feed and Explore are functional per the specs below.
  - Saved and Account can be simple placeholder screens for now (e.g.
    "Coming soon") — Don't skip them; just keep them minimal.

Burger menu (slide-out drawer, opened from a menu icon in the top bar):
  - Lists all 7 categories, each with a distinct icon:
    Business, Entertainment, General, Health, Science, Sports, Technology
  - Tapping a category closes the drawer and filters the Feed to show
    Only articles where the category matches the selected value.
  - Include an "All" option at the top of the drawer to clear the filter.
  - The active category filter should persist across the Feed until
    changed again, and be visible somewhere on the Feed screen (e.g. a
    small label or chip showing the currently selected category).

=====================================================
FEED PAGE
=====================================================

  - The top article (most recent by published_at) renders as a large
    "hero card": full-width image, ai_title (with fallback) overlaid or
    below the image, category badge, relative timestamp.
  - Below the hero, remaining articles render in a mixed card layout —
    vary between a wide horizontal card and a smaller two-column grid
    card as you scroll, so the feed doesn't feel monotonous.
  - All cards read from the articles table, respect the active category
    filter from the burger menu, and order by published_at descending.
  - Pull-to-refresh re-queries Supabase for new rows.
  - Infinite scroll / paginate (20 articles per page).
  - Tapping any card opens the article detail (full ai_content, with
    fallback, image, category, timestamp).
  - Do NOT include a share button or a "+" create/compose CTA anywhere
    on the Feed or detail view — this is a read-only news app.

=====================================================
EXPLORE PAGE
=====================================================

  - Full-screen, vertically scrollable article cards — one article
    fills exactly one viewport (1 viewport : 1 news article), snapping
    cleanly to each card as the user swipes/scrolls (like a stories or
    reels-style feed, but for reading).
  - Each full-screen card shows: title (ai_title with fallback), a short
    summary (truncated ai_content/original_content), a category badge,
    and a "View Full Article" button/CTA that opens the full detail view
    (or expands the card to show full ai_content).
  - Also respects the active category filter set from the burger menu.
  - No share button, no "+" CTA here either.

=====================================================
DESIGN SYSTEM
=====================================================

  - Display/heading font: Libre Franklin (bold weights for titles/headers)
  - Body font: Inter (regular/medium weights for body copy, captions, nav labels)
  - Primary accent color: a red in the #D85A30 range (warm red-orange) —
    use it for the active nav tab, category badges, CTAs like "View Full
    Article", and the selected state in the burger menu drawer.
  - Neutral palette (backgrounds, text, borders) should stay simple and
    light so the red accent stands out without feeling overused.
  - Category badges should each get a consistent color/icon per category
    so the same category always looks identical across Feed, Explore, and
    the burger menu.

=====================================================
RESPONSIVENESS
=====================================================

  - The entire UI must be adaptive and built mobile-first: single-column
    layouts, thumb-reachable bottom nav, comfortable tap targets, and
    the Explore page's full-viewport cards must genuinely fill the
    screen height on real mobile viewports (not just desktop preview).
  - Test that the hero card, mixed feed cards, and full-screen Explore
    cards all reflow correctly on narrow (~375–430px) mobile widths.

Do not add authentication or a login screen — this is a public read-only
news feed. Do not add write/insert functionality from the app; all writes
happen from the backend automation.
